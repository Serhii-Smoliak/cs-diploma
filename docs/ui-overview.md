# Огляд інтерфейсу CyberTactics

Документ описує **користувацький інтерфейс** платформи CyberTactics: що бачить гравець або адміністратор на кожному екрані і які дії доступні. Технічні деталі (API, БД, архітектура) — у [documentation.md](./documentation.md).

Скріншоти зберігаються в [img/](./img/) під іменами `img_37.png` … `img_61.png`.

---

## Загальні елементи

### Мова інтерфейсу

![](img/img_61.png)

- Перемикач **UA / EN** у верхній панелі змінює мову всього інтерфейсу.
- Вибрана мова зберігається для наступних сесій.

### Сповіщення

![](img/img_55.png)

- Іконка дзвоника показує кількість непрочитаних сповіщень.
- У списку — нові звання, публікації новин, відповіді підтримки тощо.
- Кнопка **«Позначити всі»** знімає позначку «нове» з усіх записів.

---

## Вхід і профіль

### Вхід / реєстрація

![](img/img_48.png)

- Форма **електронної пошти** та **пароля** для входу.
- Посилання **«Реєстрація»** для створення нового облікового запису.

### Профіль

![](img/img_46.png)

- Аватар, ім’я користувача та email.
- **«Змінити фото»** — завантаження та обрізка аватара.
- Поточний **ранг** і **досвід (XP)**.
- Кнопка **«Вийти»** завершує сесію.

## Місії та ігровий процес

### Каталог місій

![](img/img_37.png)

- Список доступних **кампаній** з описом сценарію.
- Для кожної місії — техніки MITRE ATT&CK, рівень складності, статус проходження.
- Клік по картці відкриває список завдань місії.

### Завдання місії

![](img/img_57.png)

- Ліва колонка — **послідовність завдань** (завершене / поточне / заблоковане).
- Права колонка — опис обраного завдання: навчальні цілі, тип, XP, стелс.
- Кнопка **«Почати завдання»** запускає ігровий екран.
- Блок **«Етапи атаки»** показує Kill Chain місії.

### Kill Chain

![](img/img_60.png)

- Візуальна схема **етапів атаки** (від розвідки до впливу).
- Клік по етапу показує, як поточна техніка MITRE вписується в ланцюжок.

### Типи завдань у грі

**Regex** — аналіз текстових даних регулярним виразом:

![](img/img_49.png)

- Перегляд вхідних даних (наприклад, LDAP-фрагмент).
- Поле для regex-шаблону та кнопка **«Виконати»**.

**Тактичний вибір** — один правильний варіант зі списку:

![](img/img_50.png)

- Радіокнопки з варіантами відповіді.
- Підтвердження кнопкою **«Виконати»**.

**Конструктор фішингу** — збір листа з фраз і вкладень:

![](img/img_51.png)

- Слоти для теми та тіла листа; клік по фразі додає її, клік по слоту — прибирає.
- Вибір вкладень (документ, exe, ярлик).
- **«Надіслати лист»** перевіряє збірку.

**Редактор коду** — написання команди за умовою:

![](img/img_52.png)

- Умова завдання у вигляді коментарів.
- Поле вводу коду та **«Виконати»**.

**Діалог і контекст** — супровід сценарію координатором:

![](img/img_53.png)

- Журнал повідомлень **[Система]** / **[Координатор]**.
- Поточна техніка MITRE та підказки до дії.
- Після успіху — підтвердження і приклад правильної відповіді.

**Вибір шляху реєстру** (persistence):

![](img/img_54.png)

- Список ключів Windows Registry; один варіант на вибір.

## Навчання та прогрес

### Матриця навичок (MITRE ATT&CK)

![](img/img_38.png)

- Загальний **прогрес освоєння** технік (освоєно / усього).
- Пошук за ID, назвою, описом або тактикою.
- Фільтр (усі / освоєні / неосвоєні) та групування за **тактиками MITRE**.
- Клік по картці техніки відкриває детальне модальне вікно.

### Картка техніки MITRE

![](img/img_39.png)

- Опис техніки, тактика, статус **«Освоєно»**.
- Блок **«Як це працює?»** — схема мета → дія → результат.

![](img/img_40.png)

- Приклади використання в атаках.
- Рекомендації **«Як захиститися?»**.
- Місії, де вивчається техніка; посилання **«Відкрити на MITRE»**.

### Таблиця лідерів

![](img/img_41.png)

- Рейтинг гравців за **досвідом**, кількістю **рівнів** і **технік MITRE**.
- Медалі для топ-3; рядок поточного користувача виділено.

### Звання

![](img/img_47.png)

- Шкала від **Скрипт-кіді** до **Елітного хакера** з діапазонами XP.
- Поточне звання позначено міткою **«Ваше звання»**.

## Підтримка та адміністрування

> Екрани нижче доступні лише користувачам з роллю **ADMIN** (окрім звернень користувача на `/support`, без окремого скріншота).

### Звернення (адмін)

![](img/img_43.png)

- Список тікетів зі статусами **Відкрито** / **Закрито**.
- Перегляд листування, **«Надіслати відповідь»**, **«Закрити звернення»**.

### Користувачі (налаштування)

![](img/img_44.png)

- Таблиця користувачів: роль, досвід, статус.
- **«Заблокувати»** / **«Розблокувати»** з указанням причини блокування.

### MITRE ATT&CK (синхронізація)

![](img/img_45.png)

- Кількість технік у базі та статистика **перекладів** (UK / EN).
- Кнопка **«Синхронізувати техніки»** оновлює дані з MITRE.

### Керування новинами

![](img/img_56.png)

- Список публікацій з можливістю **видалити**.
- Форма **нової публікації** (заголовок і текст UK + EN), прапорець **«Опублікувати»**, **«Зберегти»**.
